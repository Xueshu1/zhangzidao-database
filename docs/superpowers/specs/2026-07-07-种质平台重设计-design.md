# 黄渤海海珍品·种质资源数字平台 — 重设计设计文档

> 日期：2026-07-07　·　对象：`index.html`（单文件成品站重构）
> 定位：从「电影感滴动叙事页」重设计为「专业种质资源数据平台」
> 视觉基调：**A 严肃平台骨架 + B 炫技科技视觉**的混合（甲方："科技公司要有科技公司的炫技资本"）

---

## 0. 目标与约束

**甲方痛点（本次要解决的）：**
1. 滴动式（全屏滚动劫持）不适合资源库平台 → 改为专业平台布局。
2. 封面/首页不够高端大气 → 重构首页气质。
3. 全站字太小、太细、不和谐 → 排版系统升级。
4. 导航栏要重新设计；"差异性"该取消；未做的模块显示"待完善"。
5. 排布逻辑与交互逻辑要更合理严谨。
6. 除首页外其他页背景全黑，缺科技感 → 全站科技背景系统。

**硬约束：**
- 单文件 `index.html`，双击即用、离线可用、无需服务器（保持现状优势）。
- 所有数据仍为 DEMO 示例数据，保留免责声明。
- 品种真实照片沿用 `assets/species/*.jpg`，SVG 霓虹兜底不删。
- 中文优先，保留繁荣的地理专名（如"北黄海冷水团"）。

**非目标（本次不做，YAGNI）：**
- 不接后端、不做真实检索引擎（"数据检索"做成建设中占位）。
- 不引入构建工具/框架/打包（保持纯原生 + CDN）。
- 不做多语言实装（顶栏语言项可占位）。

## 1. 整站骨架：专业数据平台外壳

**废弃**：Lenis 平滑滚动劫持 + 逐屏 `data-morph` 锁定 + 底部命令搜索栏为主入口。
**改为**：固定顶栏 + 多段内容视图 + 密集页脚的常规平台结构。

```
┌─ 固定顶栏（Logo · 主导航 · 全局检索 · 语言）──────────── sticky
│
│  视图区（单文件内多"路由视图"，由 hash 切换）
│   • #home        首页（默认，多段落）
│   • #resources   种质资源（品种矩阵 + 详情）
│   • #databases   数据库总览
│   • #analysis    遗传分析
│   • #search      数据检索（建设中占位）
│   • #standards   标准规范
│   • #about       关于
│
└─ 密集机构页脚（关于/联系/共建单位/备案/免责）
```

**路由模型（关键设计）：**
- 单文件内用 hash 路由（`#home` / `#databases` …）。每个视图是一个 `<section class="view">`，切换时显示目标视图、隐藏其余，`scrollTo(0,0)`，更新顶栏高亮。
- 好处：可深链、可后退/前进、可 Ctrl-F（当前视图内）、可回顶——满足"交互严谨"。
- 保留 GSAP/ScrollTrigger 的**入场淡入**（元素进入视口时 fade-up），但**去掉滚动锁定**。视图内是普通自由滚动。
- 弹窗（品种档案/数据库详情）保留现有 modal 机制（已修好的 fixed 定位）。

**兼容与降级：**
- `prefers-reduced-motion` → 关闭所有动画，纯静态可用。
- 无 JS 兜底：视图默认全部可见（渐进增强），至少内容不丢。

**风险与对策：**
- Three.js 粒子场保留为**全站固定背景层**（`position:fixed; z-index:0`），不再随屏 morph，改为缓慢常驻动效——这是"炫技"的主要载体之一。若设备性能不足或 WebGL 不可用，回退到纯 CSS 背景栈（见 §5）。

## 2. 导航栏重构

**顶栏结构：**
```
[● 海珍·种质平台]   首页  种质资源  数据库  遗传分析  数据检索  标准规范  关于   [🔍 全局检索]  [中/EN]
```

**7 项导航与状态：**

| 导航项 | hash | 徽章 | 内容来源 |
|---|---|---|---|
| 首页 | `#home` | — | 重构首页 |
| 种质资源 | `#resources` | 已上线 | 六大主营 + 战略储备（现有 SPECIES/RESERVE 数据） |
| 数据库 | `#databases` | 部分上线 | 活体库/基因库/SNP库（现有）；转录组等标建设中 |
| 遗传分析 | `#analysis` | 试运行·示例 | 群体分化/Fst/溯源（原"差异性"升级并入） |
| 数据检索 | `#search` | 建设中 | 全局/高级检索占位页 + 规划说明 |
| 标准规范 | `#standards` | 整理中 | 框架页，标"整理中"，暂不填真实国标编号 |
| 关于 | `#about` | 已上线 | 平台概况/共建单位/备案/免责 |

**占位徽章档位：**
- `已上线`（青 `--cyan`）：功能与数据可用。
- `试运行·示例数据`（琥珀 `--amber`）：可看，数据为 DEMO。
- `整理中` / `建设中`（灰 `--ink-faint`）：进入后显示"规划说明/整理中"页，不做 404、不隐藏。

**变更点：**
- **删除**遗留"差异性"导航/命令项，语义升级为"遗传分析 → 群体分化"。
- 底部命令搜索栏（nav-cmd）**移除**：全局检索统一放顶栏，减少交互歧义。相关 JS（TMAP 命令映射、chips）一并清理。

## 3. 首页分段栈

自上而下（`#home` 视图内）：

1. **Hero 带**（约 78vh，非全屏）
   - 背景：代码生成的海洋光栅 + 基因螺旋动态（Three.js 粒子场 / Canvas，非视频文件）。
   - 主标题：H1（权威、克制），一句使命陈述。
   - 突出的全局检索框 + 示例查询 chips（`虾夷扇贝` `保藏号` `SNP`）。
   - 主 CTA：进入平台 / 浏览种质资源。
2. **统计看板（stat band）**
   - 4 个大号**等宽数字动画计数**：保藏物种 / 活体份数 / 基因组数 / SNP 位点。
   - 进入视口时 count-up 动画。tabular-nums。
3. **核心入口 Bento**
   - 6 模块卡：数据检索 · 遗传分析 · 数据汇交 · 可视化 · 表型/性状库 · 数据下载。
   - 一个旗舰大 tile + 其余 2×2；未做的标"建设中"。
4. **种质资源栅格**
   - 六大主营品种大卡：真实照片 + 拉丁名 + **数据完备度点亮矩阵**（活体✓ / 基因组✓ / SNP○ / 群体分化·建设中）。
   - 战略储备品种统一"储备中"轻量清单。
   - 点卡片 → 品种档案弹窗（沿用现有 modal + 图表）。
5. **数据库总览**
   - 活体 / 基因 / SNP / 遗传分析 四库均一资源卡（名称 + 计数 + 实时状态点 + hover 抬升）。
6. **精选物种编辑位**
   - 单一核心品种（如虾夷扇贝）编辑式展示：高清图 + 斜体拉丁名 + 关键统计。慢速、手动优先，不快速轮播。
7. **动态 / 文档**（占位，低视觉权重）
   - 两栏带日期文本列表，标"建设中"。
8. **共建单位权威墙**
   - 獐子岛集团 × 辽宁省海洋动物免疫学与疫病防控重点实验室等背书。
9. **密集机构页脚**
   - 关于/联系/共建单位/数据标准/ICP·公安备案（占位）/版权/DEMO 免责。

## 4. 排版系统（修复"太小/太细"）

比例 ≈ 大三度 1.25（信息密集页勿用黄金比，标题过大浪费竖向空间）。

| 角色 | 字号 | 字重 | 备注 |
|---|---|---|---|
| Hero H1 | `clamp(40px, 6vw, 72px)` | 700 | 字距 -0.01~-0.02em |
| H2 段标题 | `clamp(28px, 4vw, 36px)` | 600 | |
| H3 子标题 | 22–24px | 600 | |
| **正文** | **16–18px** | **400（不用 300）** | 深色低 DPI 下 300 发虚 |
| 说明/标签 | 12–14px | 400–500 | 大写 + 字距，减少纯装饰小字 |
| 统计数字 | `clamp(40px, 5vw, 64px)` | 700 | **等宽/表格数字** + count-up |

- 高级感来自**字重对比（400 vs 700）+ 留白**，非发丝细体。
- 行高：正文 1.5–1.6，标题 1.1–1.2。
- 全站字号 ≤5 种档位。中文正文用清晰黑体，不用细体。
- 全局把现有 `font-weight:300` 正文改为 400；`10–12px` 的装饰性 mono 小字提到 12px 起或删减。

## 5. 科技背景系统（"处处带科技色彩" + 可读）

**根因**：现有 `.veil`（`rgba(4,6,13,.9)` 90% 黑）与 `.solid`（`.93` 93% 黑）把 z-index:0 的粒子场盖死，只有封面透光。**全部替换**为分层背景栈。

**全站固定分层栈（fixed，z-index:0，每视图共享）：**
```
① 深蓝黑基色 #070b14（不用纯黑）
② Three.js 粒子/基因场（炫技主载体，缓慢常驻，WebGL 不可用则跳过）
③ 径向渐变 mesh：2–4 个大半径低饱和青/紫光斑
④ 蒙版淡出蓝图网格：rgba(120,180,220,.05)，64px，mask-image 径向淡出
⑤ SVG feTurbulence 颗粒噪点：opacity .035，mix-blend-mode:overlay
⑥ vignette 暗角
```

**内容层可读性（替换黑遮罩）：**
- 内容坐**半透明毛玻璃面板**：`background:rgba(18,26,42,.55); backdrop-filter:blur(14px); border:1px solid rgba(120,180,220,.12)`。
- 数据密集区（表格/图表）面板 alpha 提到 .70（更实、更易读）；概览/hero 区 .40（更透、更有氛围）。
- 仅 hero 标题正下方加**局部径向 scrim**，不全屏铺黑遮罩。
- 正文对比度始终 ≥ 4.5:1。

**炫技点缀（B 的视觉资本，施加在背景/交互层）：**
- 粒子场缓慢流动 + 鼠标力场（保留现有 Scene 能力，去掉逐屏 morph 锁定）。
- hero 局部：DNA 螺旋 / 极光光带，慢速 20–40s，`blur(80px)`，opacity ≤ .28。
- 卡片 hover：辉光描边、accent 扫过、轻微抬升。
- 统计数字 count-up；库状态"实时"脉冲点。
- 只动 `transform / opacity / filter`（GPU 友好）。

**配色纪律（严肃 vs 蹦迪的关键）：**
- 主色青 `#22d3ee`（海洋）、次色紫 `#8b7bf0`（基因）、稀有暖 `#f5a524`（仅警示/建设中）。
- **背景低饱和（鲜艳值的 15–30%）；只有数据与交互元素上高饱和。**
- `prefers-reduced-motion` → 全部动效关闭，保留静态分层背景。

## 7. 实现单元划分（便于分步与测试）

1. **外壳层**：顶栏 + hash 路由 + 视图容器 + 页脚。可独立验证"点导航切视图、深链、回退"。
2. **背景系统**：固定分层栈 + 粒子场重构（去 morph 锁定）+ 毛玻璃面板类。可独立验证"每视图有科技光 + 文字清晰"。
3. **排版系统**：CSS 变量与全局字号/字重升级。可独立验证对比度与层级。
4. **首页段落**：hero / 统计带 / bento / 品种栅格 / 库总览 / 精选 / 动态 / 权威墙。逐段接入。
5. **子视图**：种质资源 / 数据库 / 遗传分析 / 数据检索占位 / 标准规范 / 关于。逐视图接入。
6. **弹窗与图表**：复用现有，接入新数据映射。

每单元职责单一、接口清晰（视图 = hash + DOM 容器；背景 = 纯展示层；数据 = 现有常量），可独立理解与验证。
